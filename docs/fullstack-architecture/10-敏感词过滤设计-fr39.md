# 10. 敏感词过滤设计 (FR39)

## 10.1 双向过滤架构

```
输入侧:
  用户消息 → SensitiveWordFilter.checkInput(text)
    ├── 命中 → 拒绝处理，返回安全提示
    └── 通过 → 进入RAG Pipeline

输出侧:
  LLM生成内容 → SensitiveWordFilter.checkOutput(text)
    ├── 命中 → 替换为安全回答
    └── 通过 → 正常返回
```

## 10.2 实现方案

- 使用 **DFA（确定有限自动机）** 或 **Aho-Corasick** 算法实现高性能多模式匹配
- 敏感词库从 `system_configs` 表加载，Redis 缓存
- 运营后台可增删改敏感词，修改后自动刷新缓存
